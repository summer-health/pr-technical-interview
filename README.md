# PR Technical Interview

## Overview
This repository contains a technical interview exercise focused on PR (Pull Request) reviews. The application is a Provider Availability Management System with deliberately introduced issues for discussion. The system only allows adding provider's availability.

## Features
- Add provider availability
- View provider availability

## Expectations for this feature
- The system should allow users to add availability for providers.
- There should be no duplicate availability entries.
- There should be no overlapping availability entries.
- The user experience should be smooth and intuitive.

## Getting Started

### Prerequisites
- Node.js (v14+)
- npm or yarn

### Setup
```bash
npm install
```

### Start the application
```bash
npm start
```
